PatentForge is an open-source, self-hosted research tool that helps inventors organize their thinking before consulting a patent professional. Six-stage AI analysis structures your invention, discovers related prior art, and identifies the questions worth exploring with qualified counsel.

Get Started User Manual

Quick Start

git clone https://github.com/scottconverse/patentforge.git
cd patentforge
cd backend && npm install && cd ..
cd services/feasibility && npm install && cd ../..
cd frontend && npm install && cd ..
cd backend && npx prisma migrate deploy && npx prisma generate && cd ..
# Start: PatentForge.bat (Windows) or start each service manually
# Open http://localhost:8080

What PatentForge Does

Structured Technical Analysis

6-stage AI pipeline restates your invention in patent terminology, searches for related art, identifies potential issues, and organizes findings into a structured report.

Prior Art Discovery

Automated patent search via PatentsView API with AI-powered query extraction and relevance scoring. Results feed directly into the analysis stages.

Real-Time Streaming

Watch the AI write its findings in real time. Stage progress indicators show exactly where you are in the pipeline.

Cost Transparency

See the estimated API cost before you run. Dynamic pricing from LiteLLM. Per-stage token tracking after completion.

Self-Hosted & Private

Runs entirely on your machine. Invention data stays local except for the Anthropic API call. Critical for pre-filing confidentiality.

Attorney-Ready Exports

Download findings as styled HTML, Word (.docx), or Markdown. Bring structured research to your first attorney meeting instead of loose notes.

PatentForge is a research and preparation tool, not a legal service.
It does not provide legal advice, patent opinions, or attorney services. The output is designed to help you prepare for a consultation with a registered patent attorney or patent agent — not to replace one. Always seek qualified legal counsel before making patent filing decisions.

Legal guardrails built in: first-run clickwrap agreement, API key usage disclosure, and persistent disclaimers on every generated report.

Architecture

Three-service federated architecture. Each service is independently deployable. SQLite for development, PostgreSQL for Docker deployment.
