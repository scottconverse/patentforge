# v0.8 UI Polish — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Decompose the 1,486-line ProjectDetail.tsx into focused hooks and components, add sidebar status badges, fix sidebar truncation and mobile layout, and verify the sticky cancel button.

**Architecture:** Extract 6 files from ProjectDetail.tsx: 3 custom hooks (useFeasibilityRun, useRunHistory, useProjectDetail) and 3 components (ProjectSidebar, ProjectOverview, RunHistoryView). The parent component becomes a thin coordinator that owns viewMode dispatch and delegates to extracted pieces. Status badges (#7) are built into the new ProjectSidebar. Mobile sidebar uses a collapsible accordion pattern.

**Tech Stack:** React 18, TypeScript, Vitest + @testing-library/react, Tailwind CSS

**Execution order:** Sequential — each extraction changes line numbers for subsequent tasks. Tasks 1-3 extract hooks, Tasks 4-6 extract components, Task 7 adds status badges and sidebar polish, Task 8 verifies sticky cancel, Task 9 is browser QA, Task 10 is final verification.

**CRITICAL:** Every task that modifies frontend components requires the implementer to read the CURRENT state of the file before editing (CLAUDE.md rule 1). Line numbers in this plan are from the pre-decomposition state and will shift after each extraction.

- [ ] **Step 3: Implement status badges (#7)**

Each action button (Prior Art, Claims, Compliance, Application) gets a `StatusBadge` inline:

```tsx
function StatusBadge({ status, count }: { status?: string; count?: number }) {
  if (!status || status === 'NONE') return null;
  if (status === 'RUNNING') {
    return <span className="w-3 h-3 border-2 border-blue-400 border-t-transparent rounded-full animate-spin" />;
  }
  if (status === 'COMPLETE') {
    return (
      <span className="flex items-center gap-1.5">
        <span className="w-2 h-2 bg-green-500 rounded-full" />
        {count != null && count > 0 && (
          <span className="text-xs bg-green-900 text-green-300 px-1.5 py-0.5 rounded-full">{count}</span>
        )}
      </span>
    );
  }
  if (status === 'ERROR') {
    return <span className="w-2 h-2 bg-red-500 rounded-full" />;
  }
  return null;
}
```

Apply to each button:
- Prior Art: `<StatusBadge status={priorArtSearch?.status} count={priorArtSearch?.results?.length} />`
- Claims: `<StatusBadge status={claimDraftStatus?.status} count={claimDraftStatus?.claims?.length} />`
- Compliance: `<StatusBadge status={complianceStatus?.status} />`
- Application: `<StatusBadge status={applicationStatus?.status} />`
