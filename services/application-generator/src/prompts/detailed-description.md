# Detailed Description of the Preferred Embodiments

Generate the "Detailed Description" section for a patent application.

This is the longest and most important section. It must:
1. Open with a general introduction to the embodiments being described (1 paragraph)
2. Describe at least one preferred embodiment in full technical detail (multiple paragraphs)
3. Describe alternative embodiments where applicable
4. Reference figures (FIG. 1, FIG. 2, etc.) even if conceptual
5. Use reference numerals consistently for system components
6. Provide enough detail that a person skilled in the art could reproduce the invention
7. Support every element of every claim (enablement requirement under 35 USC 112(a))

Structure:
- Start broad (system overview), then go detailed (component by component)
- For each component: describe what it is, how it works, and how it interacts with other components
- Include specific examples, parameters, or ranges where available
- Describe the method/process flow if applicable

The specification language from the claim drafter should be incorporated and expanded upon — do not simply repeat it, but use it as a foundation and add significantly more detail.

Do NOT:
- Use claim-style language (no "comprising", "wherein" constructions)
- Leave any claim element unsupported by the description
- Be vague — specificity enables broad claims
